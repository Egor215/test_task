Tomato Greenhouse System (Test Task)

Система теплиц для SAMP, написанная в рамках тестового задания. Основной упор сделан на производительность при 1000+ игроках и чистоту кода.


Технический стек
MySQL R41+ (асинхронные запросы, чтобы не вешать основной поток).
Streamer Plugin (динамические объекты и зоны).
DC_CMD (быстрый командный процессор).


Как это работает (Логика)
Расчет роста: Не стал использовать переменные-счетчики в таймере. Вместо этого в БД сохраняется start_time (Unix timestamp). Прогресс считается «на лету» через разницу во времени. Это исключает баги, если сервер лагает или таймер срабатывает не вовремя.


Оптимизация стримера: Самое важное для 1000 онлайна — ящики и 3D тексты создаются с аргументом playerid. То есть стример не гоняет данные о 5000 теплиц всем игрокам в округе, а показывает объекты только их владельцу. Это экономит лимиты и не фризит клиент.


Визуализация: Выбрал объект вагончика (3169), так как он больше подходит под стиль Ранчо. Урожай отображается стопкой ящиков (1271) рядом. Ящики ставятся друг на друга через синус/косинус, чтобы они всегда были сбоку от входа, независимо от того, как игрок повернул вагончик при установке.


Нагрузка на CPU: Глобальный таймер Gh_GlobalTick делает тяжелые операции (пересоздание объектов) только тогда, когда у теплицы меняется стадия (раз в 2 минуты). В остальное время он просто проверяет условия.


Установка
Залить таблицу из database.sql в свою БД.
Прописать данные от MySQL в MYSQL_HOST, USER, PASS, DB.
Скомпилировать мод (нужен плагин a_mysql, streamer, dc_cmd).


Использование
/creategh — ставит теплицу прямо перед собой (сохраняет координаты и угол поворота).
ALT рядом с теплицей — открывает меню.
